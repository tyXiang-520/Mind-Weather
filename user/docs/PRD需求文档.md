# 心晴（MindWeather）PRD 需求文档

## 1. 产品概述

### 1.1 产品定位

心晴（MindWeather）是一款基于"天气隐喻"的情绪气象可视化平台。将用户的文字心情映射为气象状态，结合3D校园地图实现情绪空间可视化，以"用天气说心情"降低情绪表达门槛。

### 1.2 产品愿景

"让表达情绪不再需要勇气，让分享心情像分享天气一样自然。"

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|----------|
| 情绪表达门槛高（羞耻感、怕被贴标签） | 天气隐喻——用"图书馆下雨了"代替"我很难过" |
| 缺乏实时感知（不知道身边人的情绪状态） | 首页3D公共情绪地图——上下双层可视化呈现各区域情绪态势 |
| 情绪记录形式单一（散乱文本无可视化） | 个人2D气象地图 + 天气日历 + 情绪趋势分析 |

### 1.4 目标用户

在校大学生——面临学业/就业压力，有情绪表达需求但不愿直接暴露负面情绪，偏好轻量化趣味化交互。

## 2. 产品架构

### 2.1 页面结构

```
心晴 MindWeather
│
├── 首页 ─ 3D公共校园情绪地图（只读·聚合全校数据）
│   ├── 上层：天气粒子效果层（云层/雨/晴/雷等实时特效）
│   ├── 下层：3D校园地图层（Three.js + 腾讯地图）
│   ├── 区域天气实时更新（全校投稿聚合计算）
│   └── 旋转/缩放交互，点击区域查看详情
│
├── 个人页 ─ 个人气象空间（投稿入口 + 数据展示）
│   ├── 个人2D气象地图（点击区域 → 弹出入录界面 → 投稿）
│   ├── 今日个人天气
│   ├── 投稿历史列表
│   └── 情绪趋势 + 气象日历
│
└── AI情绪识别引擎
    ├── 文本情绪分类
    ├── 情绪 → 天气映射
    └── 情绪标签生成
```

### 2.2 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue3 + Vite (H5 Web) |
| 3D渲染 | Three.js |
| 2D地图 | 腾讯地图 API |
| AI | DeepSeek API (主) / 通义千问 API (备) |
| 数据库 | MySQL 8.0 |
| 后端 | Node.js (Koa) / Python (FastAPI) / Go (Gin) |

| F-01 | 邮箱注册 + 密码登录 | P0 |

### 3.2 情绪投稿 (P0)

> 用户在**个人页2D地图**点击目标区域 → 弹出文字输入框 → 提交 → AI识别 → 个人天气更新 + 首页3D地图实时刷新。首页3D地图仅查看，不提供投稿入口。

| ID | 需求 | 优先级 |
|----|------|--------|
| F-04 | 个人2D地图渲染（腾讯地图，6个区域可点击） | P0 |
| F-05 | 点击区域弹出文字投稿界面（≤500字） | P0 |
| F-06 | 提交后展示天气映射结果 + 情绪标签 | P0 |

### 3.3 AI情绪识别引擎 (P0)

| ID | 需求 | 优先级 |
|----|------|--------|
| F-07 | 文本情绪多分类（喜/怒/哀/焦虑/压力/平静等） | P0 |
| F-08 | 情绪→天气映射（开心→☀️、焦虑→☁、压力→🌧等） | P0 |
| F-09 | 情绪标签自动生成（3-5个关键词） | P0 |
| F-10 | 区域天气定时聚合计算（每30分钟刷新） | P0 |

**天气映射规则**

| 情绪 | 天气 | 图标 |
|------|------|------|
| 开心/愉悦 | 晴天 | ☀️ |
| 平静/满足 | 多云 | ⛅ |
| 焦虑/紧张 | 阴天 | ☁ |
| 压力/疲惫 | 小雨 | 🌧 |
| 低落/难过 | 大雨 | ⛈ |
| 愤怒/烦躁 | 雷暴 | 🌩 |
| 期待/希望 | 彩虹 | 🌈 |
| 孤独/迷茫 | 雾霾 | 🌫 |

| F-13 | 上层天气粒子效果（云层飘动、下雨、闪电、晴天光照） | P0 |
| F-14 | 区域天气图标实时更新 | P0 |

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标 |
|------|------|
| 投稿响应 | ≤ 2秒 |
| AI识别耗时 | ≤ 3秒 |
| 3D地图首屏加载 | ≤ 5秒 |

### 4.2 安全

- 密码bcrypt加密 · JWT鉴权 · HTTPS · 防SQL注入/XSS

### 4.3 兼容性

- H5 Web（移动端优先），后续可扩展UniApp

## 5. 核心指标

| 指标 | 目标 |
|------|------|
| 单次投稿耗时 | ≤ 30秒 |
| AI识别准确率 | ≥ 80% |
| 3D地图加载成功率 | ≥ 99% |
