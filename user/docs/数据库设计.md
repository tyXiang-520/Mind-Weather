# 心晴（MindWeather）数据库设计

## 1. 数据库选型

| 用途 | 技术 |
|------|------|
| 结构化数据 | MySQL 8.0 |

> MVP不引入Redis，直接查MySQL。后续根据性能需求再加缓存层。

---

## 2. 表结构设计 (6张表)

### 2.1 用户表 `users`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | 用户ID |
| email | VARCHAR(255) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt加密 |
| nickname | VARCHAR(64) | NOT NULL | 匿名昵称（系统生成） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

```sql
CREATE TABLE `users` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `email` VARCHAR(255) NOT NULL,
  `password_hash` VARCHAR(255) NOT NULL,
  `nickname` VARCHAR(64) NOT NULL,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_email` (`email`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 2.2 校园区域表 `campus_areas`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT UNSIGNED | PK, AUTO_INCREMENT | 区域ID |
| name | VARCHAR(64) | NOT NULL | 区域名称 |
| code | VARCHAR(32) | UNIQUE, NOT NULL | 区域编码 |
| lat | DECIMAL(10,7) | NOT NULL | 中心纬度 |
| lng | DECIMAL(10,7) | NOT NULL | 中心经度 |
| boundary | JSON | DEFAULT NULL | 区域边界坐标 |
| sort_order | INT | DEFAULT 0 | 排序 |

```sql
CREATE TABLE `campus_areas` (
  `id` INT UNSIGNED NOT NULL AUTO_INCREMENT,
  `name` VARCHAR(64) NOT NULL,
  `code` VARCHAR(32) NOT NULL,
  `lat` DECIMAL(10,7) NOT NULL,
  `lng` DECIMAL(10,7) NOT NULL,
  `boundary` JSON DEFAULT NULL,
  `sort_order` INT DEFAULT 0,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_code` (`code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 2.3 情绪投稿表 `mood_posts`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | 投稿ID |
| user_id | BIGINT UNSIGNED | FK→users.id, NOT NULL | 作者ID |
| area_id | INT UNSIGNED | FK→campus_areas.id, NOT NULL | 关联区域 |
| content_text | TEXT | NOT NULL | 文字内容 |
| emotion_type | VARCHAR(32) | DEFAULT NULL | AI识别的情绪类型 |
| emotion_confidence | DECIMAL(3,2) | DEFAULT NULL | 置信度 |
| weather_code | VARCHAR(16) | DEFAULT NULL | 天气编码 |
| weather_icon | VARCHAR(8) | DEFAULT NULL | 天气图标 |
| content_hash | VARCHAR(64) | NOT NULL | 内容哈希（去重） |
| is_deleted | TINYINT | DEFAULT 0 | 软删除 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

```sql
CREATE TABLE `mood_posts` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT UNSIGNED NOT NULL,
  `area_id` INT UNSIGNED NOT NULL,
  `content_text` TEXT NOT NULL,
  `emotion_type` VARCHAR(32) DEFAULT NULL,
  `emotion_confidence` DECIMAL(3,2) DEFAULT NULL,
  `weather_code` VARCHAR(16) DEFAULT NULL,
  `weather_icon` VARCHAR(8) DEFAULT NULL,
  `content_hash` VARCHAR(64) NOT NULL,
  `is_deleted` TINYINT DEFAULT 0,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_created` (`user_id`, `created_at` DESC),
  KEY `idx_area_created` (`area_id`, `created_at` DESC),
  KEY `idx_content_hash` (`content_hash`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 2.4 情绪标签表 `emotion_tags`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT UNSIGNED | PK, AUTO_INCREMENT | 标签ID |
| name | VARCHAR(32) | UNIQUE, NOT NULL | 标签名 |
| usage_count | INT UNSIGNED | DEFAULT 0 | 使用次数 |

```sql
CREATE TABLE `emotion_tags` (
  `id` INT UNSIGNED NOT NULL AUTO_INCREMENT,
  `name` VARCHAR(32) NOT NULL,
  `usage_count` INT UNSIGNED DEFAULT 0,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_name` (`name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 2.5 投稿-标签关联表 `post_tags`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | 关联ID |
| post_id | BIGINT UNSIGNED | FK→mood_posts.id, NOT NULL | 投稿ID |
| tag_id | INT UNSIGNED | FK→emotion_tags.id, NOT NULL | 标签ID |

```sql
CREATE TABLE `post_tags` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `post_id` BIGINT UNSIGNED NOT NULL,
  `tag_id` INT UNSIGNED NOT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_post_tag` (`post_id`, `tag_id`),
  KEY `idx_tag_post` (`tag_id`, `post_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 2.6 区域天气快照表 `area_weather_snapshots`

用于缓存区域天气聚合结果，避免每次首页加载都实时计算。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | 快照ID |
| area_id | INT UNSIGNED | FK→campus_areas.id, NOT NULL | 区域ID |
| weather_code | VARCHAR(16) | NOT NULL | 聚合主天气 |
| weather_icon | VARCHAR(8) | NOT NULL | 天气图标 |
| post_count | INT | DEFAULT 0 | 24小时内投稿数 |
| dominant_emotion | VARCHAR(32) | DEFAULT NULL | 主导情绪 |
| top_tags | JSON | DEFAULT NULL | Top 5标签 |
| emotion_distribution | JSON | DEFAULT NULL | 情绪分布占比 |
| snapshot_at | DATETIME | NOT NULL | 快照时间 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

```sql
CREATE TABLE `area_weather_snapshots` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `area_id` INT UNSIGNED NOT NULL,
  `weather_code` VARCHAR(16) NOT NULL,
  `weather_icon` VARCHAR(8) NOT NULL,
  `post_count` INT DEFAULT 0,
  `dominant_emotion` VARCHAR(32) DEFAULT NULL,
  `top_tags` JSON DEFAULT NULL,
  `emotion_distribution` JSON DEFAULT NULL,
  `snapshot_at` DATETIME NOT NULL,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_area_snapshot` (`area_id`, `snapshot_at` DESC)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 4. 定时任务

| 任务 | 频率 | 说明 |
|------|------|------|
| 区域天气聚合 | 每30分钟 | 统计各区域24小时内投稿，计算主天气/标签/情绪分布，写入 `area_weather_snapshots` |
| 历史快照清理 | 每日凌晨 | 保留最近7天快照，删除旧数据 |

---

## 5. 数据安全

- 密码 bcrypt 加密 (salt rounds ≥ 10)
- 软删除 (`is_deleted` 标记)
- 日志不记录投稿原文，仅记录ID和哈希
